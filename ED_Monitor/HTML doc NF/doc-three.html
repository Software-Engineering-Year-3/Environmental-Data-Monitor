Hashing function Documentation

This documentation provides an overview of the code for the third issue: "Maintain high levels of system security and data protection".

Overview

This issue is fundamental whitin an application that handles fragile data like personal user passwords to ensure every user's data is protected throughly.

Key Components

  • SaltyPassword:This method implements the PBKDF and SHA512 algorithms to securely hash and salt passwords.
  • VerifyPassword:To verify if the input password matches the stored password, this method ensures the check is done safely and throughly

SaltyPassword Structure

  • GenerateSalt:This section of the method generates a random salt that will be put at the start of the password to make it unique, even in case of another user chooses the same one.
  • Apply PBKDF2 algorithm: Iterating over the input password the amount if times specified in the 'iterations' variable, along side the salt generated previously and the algorithm itself, will generate the hashed password.
  • Combine salt and hashThe salt and the hashed password are combined and stored into a hashByte array.
  • Storing hashed password safely: To store the safe password, which is currently stored in a single byte array, it gets converted into a Base64 string, for better storage.

VerifyPassword Structure

  • Decode stored password:The method here decodes the stored hash, by converting it to a a single byte array, from Base64.
  • Extract salt:The first 16 bytes of the password is the salt, which gets removed to only be left with the hashed password.
  • Extract hashFrom the remaining bytes, the hash is extracted, in order to be able to check it with the input password.
  • Hash the input password with the same salt: To ensure that the check to verify the two passwords match, they both get hashed with the same algorithm, salt and iterations number.
  • Compare passwords: After the hashing of both passwords they are compared, if they match, then the user is allowed to login.
