Verify data's accuracy documentation.

This documentation provides an overview of the code for the second issue: "Verify the accuracy and integrity of collected data".

Overview

This issue is important for any users that are tasked with verifying if the data collected is accurate and integral.

Key Components

  • ViewData.xaml:The ViewData.xaml file has been slighlt modified since the previous issue. It now implements a reusable QualityDataTemplate for ensuring consistent style across all the ListViews across the code, it is easily understandable due to the consistent naming in variables and methods, to the separation in sections of the data types etc.
  • ViewData.xaml.cs:The xaml.cs file in th is case has been expanded to include methods to manually validate or flag data entries, as well as the log system to keep track of the changes made to the data.
  • IDataService.cs:IDataService defines an interface for fetching data from the mock database implementation, it also allows for easier introduction of a real database. It only focuses on data retrival keeping the interface simple and clean.
  • IQualityReading.cs:This defines the interface for all quality reading models: it allows them to have a Status property and a Logs property to track whether the data is valid and, if it has been changed, what changes have been made.
  • DataLogs.cs:DataLogs.cs defines the structure of a singular log: it has a timestamp, a record of the action performed, by who and for what reason. This was thought to ensure consistency across all the logs for all the data types.
  • Enum.cs:This simply defines the DataStatus enumeration, it has been kept separate from the rest of the code to ensure easier modification/expansion for the enumeration fields, if necessary.
  • Air/Water/WeatherService.cs:The service class for each data type shares the same code base, it has modified according to the data type it is interacting with, this helped ensure consistency, reusability and separation of concerns, since for each data type there is a class dedicated.

QualityDataTemplate Structure

  • QualityDataTemplate:Creating a base template for displaying how each item is displayed in the ListView, helpes ensure consistency and reusability. It defines the orientation of the stack layout, the padding and the fields to display, such as the SiteName, the status etc. It also presents two buttons to perform the actions to validate or flag data.

ValidateCommand Structure

  • ValidateCommand:The function to validate the data first checks if the data is valid, if it is, then it does not proceed with the function; otherwise, it sets the status to valid and creates a log entry for the action perfomed.

FlagCommand Structure

  • FlagCommand:Similar enough to the previous function, this method flags a data entry when promped, also creates a log of the action perfomed, along with all the fields specified previously.
